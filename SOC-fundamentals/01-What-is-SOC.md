# 01 – What is a SOC?

## What is a SOC?

A Security Operations Center (SOC) is a centralized team responsible for monitoring, detecting, analyzing, and responding to cybersecurity threats in an organization.

The SOC continuously monitors systems, networks, and security tools to identify suspicious activities.

---

## Main Functions of a SOC

1. Monitoring – Continuous observation of logs, network traffic, and alerts.
2. Detection – Identifying potential security threats.
3. Investigation – Analyzing alerts to determine if they are real threats.
4. Response – Taking action to contain or mitigate threats.
5. Reporting – Documenting incidents and findings.

---

## SOC Workflow

1. Alert Generation  
   Security tools (SIEM, IDS, EDR) generate alerts.

2. Alert Triage (L1 Analyst Role)  
   - Check alert details  
   - Validate if it is false positive or real threat  
   - Collect related logs  

3. Investigation (L2 / L3)  
   - Deep analysis  
   - Check affected systems  
   - Identify root cause  

4. Response & Containment  
   - Isolate system  
   - Block malicious IP  
   - Reset compromised accounts  

5. Documentation  
   - Record findings  
   - Update incident report  

---

## Role of SOC Analyst L1

- Monitor dashboards  
- Analyze alerts  
- Escalate confirmed threats  
- Document findings  
- Follow predefined playbooks  

L1 analysts are the first line of defense.
